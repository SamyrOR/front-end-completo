Webpack

É uma ferramenta amplamente utilizada para criação de bundles, ele organiza vários arquivos em um só junto de todas as suas dependências

Os 5 principais elementos do webpack:

Entry: sãos os arquivos de entradas, que possui as árvores de dependências

Output: são os arquivos de saída, local e nome do arquivo

Loaders: nativamente o webpack suporta apenas arquivos JS e JSON, porem com loaders, ele pode suportar inúmeros outros arquivos

Plugins: traz funcionalidades extras para o webpack

Mode: Desenvolvimento ou produção , diferencia o retorno dos arquivos se devem ser ja minificados como produção, ou se deve ser indentado como desenvolvimento

Estrutura

Por convenção criamos a pasta src que contem os recursos para desenvolvimento, e a pasta dist, que será o destino para os arquivos gerados pelo webpack

O principal arquivo gerado é o main.js, e na pasta raiz devemos sempre ter o arquivo webpack.config.js

Para rodar o webpack com o arquivo de configuração criado, basta rodar npx webpack --config webpack.config.js

Ou tambem podemos adicionar dentro dos scripts do package.json como "build": webpack --config webpack.config.js, e sempre que rodarmos npm run build nosso código será executado pelo webpack e criado os arquivos para produção ou desenvolvimento como descrito no arquivo de configuração
