Variáveis

É a declaração de armazenamento de algum dado na memoria do computador, que pode ser reutilizado sendo chamado com o nome declarado, exemplo de declaração variável:

var nomaDaVariavel = "Algum valor"

Os valores podem ser strings que são declaradas entre aspas, números que não precisam de aspas, undefined valor não definido(razão de muitos bugs), null elemento vazio

Temos tambem a declaração de let, que é um tipo de variavel disponivel apenas no escopo em que é declarada, e const, que recebe um valor e não pode/deve ser alterado

Podemos realizar operações matemáticas com as variáveis, com os operadores:

+ para soma

- para subtração

* para multiplicação

/ para divisão

% chamado de modulo para identificar o resto da divisão

Arrays

Array é uma variavel que carrega um conjunto de informações, ex:

var lista = ["Arroz" , "Feijão", "Salada", "Bife" , "Batata frita"]

Declarado com colchetes, cada item separado por virgula equivale a um dado, para obte-los podemos chamar a variavel informando em qual casa o item que desejamos se encontra, começando a contagem do 0

lista[0], retornará o "Arroz"

podemos manipular Arrays com algumas propriedades como

lista.push('item'), adiciona o item no final do array

lista.pop(), remove e retorna o ultimo item do array

lista.shift(), remove e retorna o primeiro item do array

lista.indexOf('item'), retorna a posição em que se encontra o item informado no parametro, caso o item não seja encontrado retorna -1

lista.join('/'), concatena todos os items do array com o caracter informado como parametro

Laços de repetição

Na linguagem de programação javascript, temos dois tipos de laços de repetição, o for e o while, os laços de repetição executam determinado bloco de código enquanto a condição for verdadeira:

O for recebe como parametro a variavel, a condição e a execução:

for(var x = 0; x < 10; x++) {

código a ser executado até que x seja > que 10.

}

O while executa a mesma operação, porem é declarado de forma diferente, ja temos que ter a variavel declarada antes

x = 0

Então executamos o código

while(x < 10) {

código a ser executado enquanto a condição seja verdadeira

}

Programação orientada a eventos

Em javascript programamos muito orientado a eventos, ou seja, se algo ocorrer estamos ouvindo o evento acontecer, e quanto acontecer executamos determinada função

Eventos de mouse:

click = quando o elemento é clicado

dblclick = acionado quando se clica duas vezes rapidamente

contextmenu = menu disposto no botão direito

mousedown = quando botão esquerdo é clicado

mouseup = quando se solta o click do botão esquerdo

mouseover = quando o mouse passar pelo elemento

mouseout = quando o mouse passa por cima e sai do elemento

mousemove = captura o movimento do mouse em cima do elemento

Eventos de teclado

Para eventos de teclado temos apenas 3 opções

keydown = quando o botão é apertado

keyup = quando eu solto o botão

keypress = enquanto o botão é pressionado (segurado)

Mas devemos capturar o botão que foi apertado para executar determinadas ações, fazemos isso passando o evento como paramentro, e usando propriedades com event.key para identificar a tecla

Podemos tambem saber se o shift, ctrl ou alt estavam pressionadas com as propriedades event.shiftKey, .ctrlKey, .altKey, todas retornam um valor booslean

Evento de página

Evento roda quando algo ocorre na página

onload = quando a página termina de ser carregada

unload = quando a página é fechada

Eventos de formulário

submit = quando o formulário é enviado

focus = quando um elemento é selecionado

blur = quando um perde a seleção

change = quando o elemento tem o sue valor trocado

Temporizadores

Para executarmos uma tarefa a cada determinado tempo, ou apenas uma vez após um certo tempo, temos:

sessionStorage() e localStorage()

Estas duas funções, nos permite gravar dados no navegador, durante apenas na sessão (até o navegador ser fechado) e localmente (mesmo que feche o navegador, desligue, etc - até ser removido manualmente)

A sintaxe é simples:

localStorage.setItem("chave", valor); para adicionarmos itens no localStorage

localStorage.getItem("chave"); recupera o valor da chave informada

localStorage.removeItem("chave"); remove a chave e seu valor

Os mesmos métodos podem ser utilizados em sessionStorage.

Spread e Rest Operator

Ambos operadores são declarados da mesma forma, com reticencias ...

Spread

O spread muito utilizado, tem como objetivo desconstruir arrays e objetos, facilitando assim a integração em uma nova variavel

let lista2 = [...lista1, 3,4,5]

Neste case a lista2 tera os itens da lista 1 junto com os outros itens informados

Rest operator

Com a mesma sintaxe do spread, porem este é utilizado apenas como parametro de uma função:

function receberNumeros(...numeros) {

Quando temos uma function que nao sabemos, ou queremos deixar não fixo o número de parâmetros que desejamos receber, podemos utiliza-lo, e passar quantos parâmetros forem na chamada da função, que ao receber com rest irá transformar todos em uma array, podendo ser utilizado spread posteriormente para decomposição da array gerada pelo rest

Programação orientada a objeto: POO

Podemos com javascript realizar programção orientada a obejto, um poderosos paradigma que estrutura o código e deixar uma manutenção e uma escrita muito mais elaborada

Para iniciarmos nesse paradigma, temos que entender as classes:

class Animal {

constructor(orelhas, pernas) {

this.orelhas = orelhas

this.pernas = pernas

}

falar(){

console.log('Hello there')

}

get qtdPernas() {

return this.pernas

}

set inserirPernas(quantidade) {

this.pernas = quantidade

}

Para definir uma classe, iniciamos com a palavra reservada 'class' antes do nome da classe e iniciamos com {

Após isso definimos a função construtora, construct () que pode receber parâmetros que vão ser definidos na instância da classe, e abrimos com {

Ainda dentro do constructor definimos os atributos com this.nomedoatributo que vão ser elementos do objeto

Fora do constructor podemos definir métodos, que basicamente são funções que ao serem invocadas executam o seu código

E ao final da classe definimos os getters e setters, que basicamente são propriedades do objeto instanciado que vai recuperar os dados da classe no getter e definir no setter

Junto com os módulos, o torna uma poderoso paradigma de programação avançada

Geralmente instanciada como return de uma function que tem que aguardar a resposta de alguma requisição

functino fazerAlgo(){

return new Promise((resolve, reject) => {

bloco de código a ser executado

resolve(parametro da resposta)

if (error) reject(error)

})}
