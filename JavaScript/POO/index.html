Pilares da programação orientada a objetos

Abstração: Isolar de um objeto somente conceitos que são necessários para o funcionamento do programa.
Encapsulamento: Ocultar partes do objeto que são úteis apenas para uso interno e exibir apenas o necessário para uso externo
Herança: Passar características de um objeto para outro.
Polimorfismo: Algo que é polimorfo, tem a habilidade de assumir diferentes formas
